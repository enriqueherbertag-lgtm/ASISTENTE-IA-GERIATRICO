# Público Objetivo - Asistente IA Geriátrico

## 1. Usuario Final (Adulto Mayor)

### Perfil Demográfico
| Rango etario | 70-95 años |
|:---|:---|
| Género | Mayoritariamente mujeres (esperanza de vida mayor) |
| Situación de vivienda | Solo, con cónyuge, o con cuidador |
| Nivel educativo | Variable, pero generalmente bajo en tecnología |
| Ingresos | Pensión básica o media |

### Condiciones de Salud Prevalentes
| Condición | Prevalencia en el grupo |
|:---|:---|
| Presbicia | 100% |
| Retinopatía diabética | 30-40% (si diabético) |
| Diabetes tipo 2 | 25% |
| Deterioro cognitivo leve | 15-20% |
| Alzheimer / demencia | 10% |
| Riesgo de caídas | 30-50% |
| Hipoacusia leve | 60% |

### Capacidades Conservadas
| Capacidad | Por qué es importante |
|:---|:---|
| Audición (aunque sea con ayuda) | El dispositivo se comunica por voz |
| Comprensión del lenguaje | Entiende instrucciones simples |
| Memoria emocional | Reconoce voces familiares |
| Rutinas automatizadas | Cocinar, caminar, tomar medicación |
| Deseo de autonomía | Quiere seguir siendo independiente |

### Limitaciones Críticas
| Limitación | Impacto en el diseño |
|:---|:---|
| No usa pantallas táctiles | Interfaz solo por voz |
| Se olvida de cargar dispositivos | Batería de larga duración (7+ días) |
| Rechaza lo "nuevo" | El dispositivo debe ser invisible |
| No recuerda instrucciones complejas | Configuración única, uso intuitivo |
| Puede perder el dispositivo | Sujeción segura, alerta de pérdida |

## 2. Cuidador Principal

### Perfil
| Relación | Hijo/a (50%), cónyuge (30%), profesional (20%) |
|:---|:---|
| Edad | 45-70 años |
| Situación laboral | Activo (hijos), jubilado (cónyuge) |
| Carga horaria | 24/7 (no profesional), turnos (profesional) |

### Necesidades
| Necesidad | Cómo la satisface el dispositivo |
|:---|:---|
| Reducir repeticiones | El dispositivo responde preguntas recurrentes |
| Dormir tranquilo | Alerta nocturna si hay problemas |
| Saber que no se cayó | Detección automática de caídas |
| Que el familiar esté orientado | Recordatorios de rutina |
| Tener un respiro | El dispositivo cubre momentos de ausencia |
| No sentirse culpable | Sabe que hay una red de apoyo tecnológico |

### Limitaciones
| Limitación | Impacto en el diseño |
|:---|:---|
| No tiene tiempo para configurar cosas | Configuración simple, una sola vez |
| No es técnico | Interfaz intuitiva, guías claras |
| Está agotado | El dispositivo debe funcionar solo |

## 3. Familia Extendida

### Perfil
| Relación | Nietos, hermanos, sobrinos |
|:---|:---|
| Edad | Variable |
| Contacto | Esporádico (visitas, llamadas) |

### Aportes al Sistema
| Aporte | Cómo se integra |
|:---|:---|
| Grabación de voz | El nieto graba mensajes para el abuelo |
| Recetas familiares | Se cargan en la base de datos |
| Fotos familiares | Para reconocimiento facial |
| Rutinas compartidas | Horarios de visitas, comidas familiares |

### Beneficios
| Beneficio | Por qué importa |
|:---|:---|
| El abuelo los reconoce | Reconocimiento facial asistido |
| Escuchan su voz | Conexión emocional diaria |
| Saben que está cuidado | Tranquilidad familiar |

## 4. Instituciones (Compradores Potenciales)

### Tipo de Institución
| Tipo | Ejemplo |
|:---|:---|
| Residencias geriátricas | Hogares de ancianos, ELEAM |
| Hospitales | Geriatría, servicios de crónicos |
| Sistemas de salud pública | Fonasa, hospitales públicos |
| Mutuales de seguridad | IST, ACHS |
| Gobiernos locales | Municipios con programas de adulto mayor |

### Necesidades Institucionales
| Necesidad | Cómo la satisface el dispositivo |
|:---|:---|
| Reducir caídas en residentes | Detección + prevención |
| Monitorear crónicos sin saturar personal | Automatización de alertas |
| Diferenciarse de la competencia | Tecnología innovadora |
| Cumplir indicadores de calidad | Datos de uso, reducción de incidentes |
| Controlar costos | Precio bajo, sin infraestructura adicional |

## 5. Segmentación por Nivel de Autonomía

### Grupo A: Autónomo con riesgos
| Características | Vive solo, activo, pero con alguna condición |
|:---|:---|
| Uso del dispositivo | Recordatorios, monitoreo de glucosa, lectura de etiquetas |
| Beneficio principal | Prevención, autonomía prolongada |

### Grupo B: Con deterioro cognitivo leve
| Características | Olvidos frecuentes, repite preguntas, desorientación leve |
|:---|:---|
| Uso del dispositivo | Memoria contextual, reconocimiento de personas, rutinas |
| Beneficio principal | Retraso de la institucionalización |

### Grupo C: Alzheimer o demencia moderada
| Características | No reconoce familiares, se pierde, agitación nocturna |
|:---|:---|
| Uso del dispositivo | Voz familiar, geocerca, orientación temporal |
| Beneficio principal | Contención, seguridad, alivio del cuidador |

### Grupo D: Institucionalizado
| Características | Vive en residencia, requiere cuidados constantes |
|:---|:---|
| Uso del dispositivo | Monitoreo continuo, alertas al personal |
| Beneficio principal | Mejor calidad de atención, menos personal necesario |

## 7. Conclusión

El Asistente IA Geriátrico no es para un nicho. Es para **cualquier persona mayor que quiera seguir viviendo con dignidad**, y para **cualquier familia que quiera acompañarla sin desmoronarse en el intento**.

Su público es tan amplio como el envejecimiento mismo.
